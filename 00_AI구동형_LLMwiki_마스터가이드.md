# AI 구동형 개인 LLMwiki 마스터 가이드

> **핵심 철학**: AI가 단순한 도구가 아니라 "지식 관리 파트너"로 작동하는 시스템을 만든다.
> 내가 정보를 던지면, AI가 요약·분류·연결·검색까지 처리한다.

---

## 시스템 개요: 무엇을 만드는가?

### 전통적 위키 vs AI 구동형 위키

| 구분 | 전통적 위키 | AI 구동형 위키 |
|------|------------|---------------|
| 정보 입력 | 내가 직접 정리해서 씀 | 원문 던지면 AI가 정리 |
| 분류·태깅 | 내가 수동으로 지정 | AI가 자동 분류 제안 |
| 검색 | 키워드 검색 | 자연어 질문으로 검색 |
| 연결 | 링크를 내가 직접 삽입 | AI가 관련 문서 자동 연결 |
| 합성 | 내가 직접 읽고 정리 | AI가 여러 문서 교차 분석 |
| 유지 비용 | 높음 (시간) | 낮음 (프롬프트만) |

### 이 시스템이 해결하는 문제

```
[현재 상황]
뉴스 / 논문 / 회의 / 대화 → 뇌에서 소멸 → 6개월 후 같은 공부 반복

[목표 상황]
뉴스 / 논문 / 회의 / 대화
        ↓ (AI 처리)
요약 → 분류 → 연결 → 검색 가능 → 의사결정에 즉시 활용
```

## PART 1 — 시스템 아키텍처

### 1-1. 5단계 지식 파이프라인

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. CAPTURE │───▶│ 2. PROCESS  │───▶│  3. STORE   │
│  (수집)      │    │ (AI 처리)   │    │  (저장)      │
└─────────────┘    └─────────────┘    └─────────────┘
                                              │
                                              ▼
┌─────────────┐    ┌─────────────┐
│ 5. SYNTHESIZE│◀──│ 4. RETRIEVE │
│  (AI 합성)   │    │  (검색)     │
└─────────────┘    └─────────────┘
```

#### CAPTURE — 수집
원문을 그대로 AI에게 전달하는 모든 방법
- URL 붙여넣기
- PDF 업로드
- 회의 메모 복사
- 음성 녹음 → 텍스트 변환 후 전달
- 스크린샷

#### PROCESS — AI 처리 (핵심 단계)
AI가 수행하는 자동 작업
- 3줄 요약 생성
- 핵심 키워드 추출
- 카테고리 자동 분류
- 관련 기존 문서 연결 제안
- 비즈니스 임팩트 한 줄 평가

#### STORE — 저장
처리된 내용을 위키 구조에 맞게 파일로 저장

#### RETRIEVE — 검색
"우리 회사 AI 전략에 RAG를 써야 할까?" → 관련 문서 즉시 소환

#### SYNTHESIZE — 합성
여러 문서를 바탕으로 새로운 인사이트 생성
예: "지금까지 수집한 정보를 바탕으로 2026년 AI 투자 우선순위 제안해줘"

## PART 3 — AI 처리 워크플로우 (실전 프롬프트 포함)

### 3-1. 정보 유형별 처리 방법

#### 타입 A: 웹 아티클 / 뉴스

**Claude에게 이렇게 요청:**
```
다음 기사를 내 LLMwiki 트렌드 분석 템플릿 형식으로 정리해줘.
중요도 판단 기준: 비즈니스 활용 가능성과 시장 파급력.

[기사 URL 또는 전문 붙여넣기]
```

#### 타입 B: 기술 논문

**Claude에게 이렇게 요청:**
```
이 논문을 C-level 의사결정자 관점에서 요약해줘.
기술적 세부사항보다 "왜 중요한가"와 "어떻게 활용할 수 있나"에 집중.
분량: A4 반 페이지 이내.
내 위키 개념_문서_템플릿 형식으로 작성해줘.

[논문 PDF 또는 초록 붙여넣기]
```

#### 타입 C: 회의 / 미팅 메모

**Claude에게 이렇게 요청:**
```
다음 회의 메모에서 AI/LLM 관련 결정사항과 인사이트를 추출해줘.
형식: 날짜 / 결정내용 / 배경 / 후속 액션
내 의사결정 로그에 추가할 수 있도록 정리해줘.

[회의 메모 붙여넣기]
```

#### 타입 D: 경쟁사 AI 동향

**Claude에게 이렇게 요청:**
```
[경쟁사명]의 최근 AI 관련 움직임을 분석해줘.
분석 항목: 1) 어떤 AI를 도입했나 2) 예상 목적 3) 우리에게 시사점
내 벤더_평가 폴더에 저장할 수 있도록 구조화해줘.
```

#### 타입 E: 유튜브 / 팟캐스트

**Claude에게 이렇게 요청:**
```
다음 내용을 핵심 포인트만 추출해서 정리해줘.
(자막 또는 스크립트 붙여넣기)
내 위키에 추가할 가치가 있는 내용인지도 판단해줘.
```

### 3-2. 주간 자동화 루틴

```
매주 월요일 아침 (30분):

① CAPTURE (10분)
   지난 주 저장해둔 링크/메모를 _inbox 폴더에 정리

② PROCESS (15분)
   Claude에게: "inbox 폴더의 내용을 처리해서 각 카테고리에 배치해줘"

③ CONNECT (5분)
   Claude에게: "이번 주 추가된 내용과 기존 문서 중 연결할 것을 찾아줘"
```

### 3-3. 강력한 합성 프롬프트

#### 인사이트 합성 요청
```
내 위키에 있는 [특정 주제] 관련 문서들을 종합해서
현재 시점에서 가장 중요한 의사결정 포인트 3개를 뽑아줘.
각각에 대해 찬반 논리와 내 추천 방향을 제시해줘.
```

#### 격차 분석 요청
```
내 위키의 현재 구조를 보고,
비어있거나 업데이트가 필요한 중요 영역을 찾아줘.
우선순위 순서로 정리해줘.
```

#### 브리핑 생성 요청
```
내일 [주제] 관련 미팅이 있어.
내 위키에서 관련 내용을 모아서 5분 안에 읽을 수 있는
사전 브리핑 문서를 만들어줘.
```

## PART 4 — 도구 스택 선택

### 4-1. 3가지 구축 시나리오

#### 시나리오 1: 즉시 시작형 (Claude Cowork만 사용)
```
난이도: ★☆☆☆☆
비용: Claude 구독료만
설정 시간: 0분 (지금 바로 가능)

[방법]
- 이 프로젝트 폴더(LLM wiki)를 Cowork에 마운트
- Claude에게 직접 요청해서 문서 생성·수정
- 파일은 내 컴퓨터에 저장됨

[한계]
- 자동화 없음, 매번 수동 요청 필요
- 모바일에서 접근 어려움
```

#### 시나리오 2: 파워유저형 (Obsidian + Claude)
```
난이도: ★★★☆☆
비용: Obsidian 무료 + Claude 구독
설정 시간: 2~3시간

[방법]
- Obsidian으로 마크다운 위키 관리
- Obsidian Smart Connections 플러그인 → 문서 간 연결
- Claude에서 처리 후 Obsidian에 복붙

[강점]
- 그래프 뷰로 지식 맵 시각화
- 로컬 저장, 완전한 통제권
- 강력한 검색 및 링크 기능
```

#### 시나리오 3: 팀 확장형 (Notion + Claude API)
```
난이도: ★★★★☆
비용: Notion Teams + Claude API 비용
설정 시간: 1~2일

[방법]
- Notion을 지식 베이스로 사용
- Claude API 연동으로 자동 처리
- 팀원과 공유, 권한 관리

[강점]
- 팀 전체가 활용 가능
- 웹/모바일 모두 접근
- 데이터베이스 뷰, 필터 강력
```

### 4-2. 핵심 도구 매핑

| 기능 | 즉시 시작형 | 파워유저형 | 팀 확장형 |
|------|-----------|-----------|---------|
| AI 처리 | Claude Cowork | Claude | Claude API |
| 저장 | 로컬 마크다운 | Obsidian | Notion |
| 검색 | Cowork 내 검색 | Obsidian Search | Notion Search |
| 시각화 | 없음 | 그래프 뷰 | 데이터베이스 뷰 |
| 자동화 | 없음 | 제한적 | 가능 |
| 팀 공유 | 불편 | 불편 | 강력 |

## PART 5 — 단계별 구축 로드맵

### Phase 1: 기반 구축 (1주차)

```
□ 폴더 구조 생성 (Claude에게 요청)
□ 템플릿 파일 3개 만들기 (개념 / 트렌드 / 의사결정 로그)
□ 현재 알고 있는 핵심 개념 5개 문서화
□ 즐겨찾는 정보 소스 목록 정리
```

**1주차 목표**: 빈 집이지만 구조가 잡힌 위키

### Phase 2: 콘텐츠 적재 (2~4주차)

```
□ 주 1회 30분 루틴 시작
□ 매주 새 문서 3~5개 추가
□ 모델 비교표 채우기 (GPT-4o, Claude, Gemini)
□ 우리 회사 AI 유스케이스 최소 2개 문서화
□ 트렌드 레이더 첫 번째 버전 작성
```

**4주차 목표**: 20~30개 문서, 검색할 내용이 생김

### Phase 3: 연결 & 심화 (2~3개월차)

```
□ 문서 간 연결 고리 만들기 (관련 문서 링크)
□ 모델 선택 기준표 완성 (실제 사용 경험 기반)
□ ROI 계산 프레임워크 실전 적용
□ 팀원과 공유 방법 결정
□ 위키 기반으로 첫 번째 전략 문서 작성
```

**3개월차 목표**: AI 관련 질문에 위키를 먼저 보게 됨

### Phase 4: 시스템화 (3개월 이후)

```
□ 자동화 루틴 정착 (수동 개입 최소화)
□ 팀 위키로 확장 or 개인 심화
□ 위키 기반 정기 브리핑 문서 생성
□ 연간 AI 전략 수립에 위키 활용
```

**6개월차 목표**: 위키가 스스로 누적·성장하는 복리 자산

## PART 6 — 품질 관리 & 함정 회피

### 6-1. 위키 신선도 점검 체크리스트 (월 1회)

```
□ 6개월 이상 된 모델 정보 업데이트했는가?
□ 더 이상 사용 안 하는 도구는 보관 처리했는가?
□ 가장 많이 참조하는 문서는 무엇인가? (가치 파악)
□ 전혀 안 보는 문서는 무엇인가? (구조 개선 힌트)
□ 비어있는 중요 주제가 있는가?
```

### 6-2. 흔한 실수 5가지

| 실수 | 증상 | 해결책 |
|------|------|--------|
| 수집 과다 | inbox가 넘침, 처리 안 됨 | 일주일에 3~5개만 선별 |
| 완벽주의 | 못 쓴 문서가 많음 | 초안 70%로 게시, 나중에 보완 |
| 복붙 위키 | 내 생각 없이 AI 요약만 가득 | 모든 문서에 "나의 코멘트" 필수 |
| 구조 집착 | 폴더만 만들고 내용 없음 | 구조보다 콘텐츠 우선 |
| 고립된 위키 | 실제 업무와 단절됨 | 미팅·의사결정 때 위키 먼저 펼치기 |

### 6-3. AI 출력 신뢰도 관리

AI가 생성한 내용은 반드시 이 기준으로 분류:

```
🟢 검증됨: 공식 문서나 내 경험으로 확인
🟡 초안: AI 요약이지만 미확인
🔴 오래됨: 6개월 이상 지나 재확인 필요
```

모든 AI 생성 요약에는 원문 링크와 생성 날짜 필수 기재.
