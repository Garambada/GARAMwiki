전철전력 설계 업무의 구조적 문제 4가지
1
데이터 단절
설계~운영 정보 고립
CAD·Excel·HWP·SCADA가 각각 독립 운영. 단계 간 데이터 연속성 없음. 수작업 교차검증에 과도한 인력 투입.
2
인적 오류 내재
반복적 실수 구조
mm↔m·EA↔SET 단위 환산 오류, 수량 중복 계상, 품셈 공종부호 부적용이 반복 발생. 검증 접점 다수 존재.
3
설계 환류 부재
운영 데이터 미활용
SCADA의 실제 부하 데이터·가동률·고장 이력이 설계에 환류되지 않음. 경험적 안전계수 의존 → 과설계·과소설계 리스크.
4
검증 체계 한계
개인 역량 의존
설계 검토가 숙련 전문가의 개인 역량에 크게 의존. 체계적 규칙 부재 → 검토 일관성·재현성 미확보.

핵심 AI 기술 3종 — 유기적 결합 아키텍처
Ontology
온톨로지 기반 지식 DB
OWL로 도메인 개념·속성·관계를 형식화. 자동 추론(Reasoning)과 논리적 일관성 검증 지원. HyDRA 프레임워크(2025)로 AI 에이전트 간 합의를 통해 자동 구축.
핵심 클래스설비·자재·도면·수량·품셈·기준
구축 방법Top-down + Bottom-up Hybrid
구축 단계CQ정의 → 설계 → 인스턴스화 → 검증
적용 근거Kaisera et al. (2025) HyDRA
GraphRAG
지식그래프 기반 검색증강생성
다단계 관계(Multi-hop) 추적으로 의미 연결 정보 종합 검색. 벡터 기반 RAG 대비 구조적 관계 추론 정확도 유의미 향상. Neo4j + ChromaDB 3중 구조.
저장소Neo4j + ChromaDB + PostgreSQL
질의 방식Cypher + 벡터 검색 동시 처리
주요 시나리오규격검증·수량추적·오류영향분석
적용 근거Ghosh & Mittal (2025) Frontiers AI
PINN
물리정보 인공신경망
물리 법칙(PDE)을 손실함수에 통합. 제한된 데이터에서도 물리적으로 타당한 예측 수행. 전력 계통·배터리·재생에너지 3대 영역 검증 완료(2026 ScienceDirect 체계적 분석).
연산 속도수치해석 대비 최대 87배 빠름
적용 영역용량산정·부하예측·전압강하·RUL
적용 단계2~3단계 (확산·고도화)
적용 근거Misyris et al. (2019) arXiv
⚡ 기술 간 연계 시너지
온톨로지가 도메인 지식의 구조적 골격을 제공 → 지식그래프가 실제 설계 데이터를 인스턴스화 → GraphRAG가 이 구조화된 지식을 LLM 추론에 활용 → PINN이 물리 법칙에 근거한 수치적 검증으로 AI 추론의 물리적 타당성 보장. 텍스트 정보(온톨로지·GraphRAG)와 수치 정보(PINN)를 동시에 검증하는 다차원 설계 검증 체계 실현.

PoC 시스템 — 4계층 아키텍처
Layer 1 — 데이터 수집
문서 업로더 · OCR · 파서
FastAPI · ezdxf · openpyxl · Tesseract · Tika
이질적 형식의 설계 문서를 구조화된 데이터로 변환. REST API 통신.
Layer 2 — 지식 관리
온톨로지 · 지식그래프 DB · 벡터 DB
Neo4j · Protégé · ChromaDB · PostgreSQL
도메인 지식의 구조화된 저장·검색·추론 기반 제공. Bolt/HTTP 통신.
Layer 3 — AI 추론
GraphRAG · 규칙 엔진 · LLM · 매칭 엔진
LangChain · LlamaIndex · SymPy · Pint · LLM API
구조적 검색·규칙 기반 검증·AI 추론을 통한 검증 실행. gRPC/REST.
Layer 4 — UI
대시보드 · 검증 뷰 · 보고서 생성
Next.js · React · Chart.js · PDFMake
사용자 역할에 맞는 시각화·상호작용 인터페이스 제공. WebSocket/REST.

R1 설계 문서 비정형성 — 프로젝트별 양식·구조 차이 상 상 문서 유형별 파싱 템플릿 라이브러리 구축, 적응형 파서 개발 파싱 성공률 ≥ 90%
R2 데이터 품질 부족 — OCR 오류, 불완전 문서 상 중 전처리 파이프라인 강화, 품질 기준 사전 합의, 수동 보정 인터페이스 데이터 품질 점수 ≥ 0.85
R3 현업 사용자 저항감 — 업무 프로세스 변경 부담 중 중 M2부터 조기 참여 유도, 단계적 도입, 챔피언 사용자 육성 SUS ≥ 70점
R4 SCADA 연계 지연 — 접근 권한·데이터 형식 문제 중 중 Mock 데이터 기반 개발 선행, 연계 인터페이스 표준화, 조기 기술 협의(M1) 연계 API 가용성
R5 LLM 환각(Hallucination) 위험 — 검증 오류 유발 상 중 RAG 기반 근거 강제, 규칙 엔진↔LLM 교차 검증, 신뢰도 점수 필터링 환각 발생률 < 5%
R6 보안 사고 — 설계 문서 데이터 유출 상 하 공단 보안규정 준수(TLS 1.3 + AES-256 + RBAC), 보안 감사 정기 시행 보안 점검 통과율 100%
