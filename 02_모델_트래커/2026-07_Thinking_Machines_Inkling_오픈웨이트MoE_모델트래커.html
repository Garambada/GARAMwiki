Inkling의 가장 핵심적인 차별점. effort를 0.2에서 0.99 사이로 설정하면 모델이 소비하는 토큰 수가 달라지고, 성능과 비용이 함께 바뀐다. RL 학습 시 시스템 메시지로 effort 레벨을 지정하고 토큰당 비용을 조정해 이 능력을 학습시켰다.

수치로 본 효율: Terminal Bench 2.1에서 Nemotron 3 Ultra와 동일 점수를 달성하는 데 약 1/3의 토큰만 소비한다. 에이전트 워크플로우에서 수백만 번 호출될 때 비용 차이는 결정적이다.

EP104에서 노정석이 언급한 "Codex effort 단계 세분화"와 동일한 트렌드 — 에이전트 시대의 AI는 강도를 조절할 수 있어야 한다.

Epistemics
에피스테믹스 — 교정·명령이행·검열거부
ForecastBench Brier 61.1 (Gemini 3.1 Pro 동등) · 검열 비준수 최상위

Thinking Machines가 별도로 정의한 모델 신뢰성 개념 "Epistemics" — 세 가지로 구성된다.

① 교정(Calibration): 답을 모를 때 "모른다"거나 확률적 추정을 제공. 예측/예보 과제에서 중요. ForecastBench no-search에서 61.1 Brier Index — Gemini 3.1 Pro와 동등. RL 학습에 proper scoring rules를 사용해 달성.

② 명령이행(Instruction Following): Rubric grader(체크리스트 채점) + Claims grader(사실 검증 — 웹 검색 기반)를 동시에 사용해 도움이 되면서 환각도 줄이는 두 마리 토끼를 잡았다. IFBench 79.8%.

③ 검열 거부(Censorship Non-compliance): Cognition의 Propaganda & Censorship Eval에서 강한 검열 비준수 패턴 — 공식 입장이 아닌 사용자 의도에 따라 답변하는 모델.

RL 대규모 학습 — 30M+ 롤아웃 · 사고의 압축

SFT 초기화 후 비동기 대규모 RL로 추론 성능을 로그-선형으로 향상. SFT Reward 0.264 → 출시 Reward 0.356(+0.092). 두 번의 장기 연속 런으로 안정적 학습 유지.

RL이 명시적으로 목표로 하지 않았는데도, Chain of Thought가 더 간결해지는 방향으로 자연스럽게 수렴했다. 문법적 오버헤드가 사라지고 핵심 추론만 남았다 — 효율이 압축을 낳은 것이다.
— Thinking Machines Lab, Inkling 릴리즈 노트 (2026-07-15)
RL 초기 — 문법적·장황한 사고
We need to understand the operator.
The 5D line element is ds² = e^{2A(x)}...
The internal coordinate is periodic.
The background is a warped product...
Wait, the ds² is e^{2A} (ds²_4d + dx²)?
RL 후기 — 압축된 텔레그래픽 사고
We need determine eigenvalue problem
for spin-2 fluctuations h_{μν}(x,y)...
Internal metric is e^{2A} dx²?
Wait internal metric is e^{2A} dx²;
warp factor same for 4d and internal? Yes.

"We need to understand" → "We need determine" — 조사·접속사가 사라지고 핵심 논리만 남는다. 동일한 답에 도달하지만 훨씬 적은 토큰. Cognition의 SWE-1.7 학습에서도 같은 패턴이 독립적으로 관찰됐다.

GARAMbada 인사이트 — Inkling이 WISE42 관점에서 갖는 전략적 의미
01
Controllable Effort = 에이전트 비용 최적화의 새 표준
GPT-5.6 Codex effort 단계(EP104), Inkling controllable effort — 같은 트렌드가 두 회사에서 독립적으로 나왔다. 에이전트가 수백만 번 호출되는 시대에 "강도를 조절할 수 있는 모델"은 비용 구조를 근본적으로 바꾼다. WISE42 KRNA·FiPA·AGP 프로젝트에서 배치 작업(문서 분류, 데이터 추출)에는 low effort, 복잡한 도메인 추론에는 high effort로 운영하는 전략이 가능하다.
02
오픈웨이트 + 파인튜닝 = 도메인 모델 구축의 현실적 경로
ICML에서의 "스케일주의 vs 도메인주의" 논쟁에서 Inkling은 도메인주의의 인프라를 공급한다. Thinking Machines가 Tinker를 파인튜닝 플랫폼으로 포지셔닝한 것 = "좋은 기반 모델 + 고객 도메인 데이터 = 특화 모델"이라는 공식. KRNA(철도 도면 추출), FiPA(공공조달 법령), Onion(OT 스택 이상감지) — 세 프로젝트 모두 도메인 온톨로지 + 파인튜닝이 답인 케이스다. Inkling의 오픈웨이트 + Tinker는 GPT/Claude API에 대한 프론티어 랩 종속 리스크를 분산하는 대안이다.
03
RL의 사고 압축 현상 — "텔레그래픽 지성"이 새 표준
RL 후기 Chain of Thought가 문법적 오버헤드 없이 핵심 논리만 남기는 압축 현상. Cognition SWE-1.7에서도 독립적으로 관찰 — 이것은 우연이 아니라 RL의 구조적 결과다. 인간의 글쓰기와 AI 추론의 간극이 커진다는 의미. 인간이 "AI 출력을 이해하기 어렵다"는 새로운 형태의 인지 부채(Geoffrey Litt)가 생긴다. Company Builder 관점: AI가 만드는 결과물을 검증·해석하는 인간의 역량이 다음 병목이 될 것이다.
04
Epistemics(교정·검열거부) = AI 신뢰성의 새 평가 축
"답을 모를 때 모른다고 말하는 AI"가 이제 명시적 학습 목표가 됐다. 공공 AX(KRNA·FiPA) 컨텍스트에서 교정된 확신(calibrated confidence)은 결정적으로 중요하다 — 틀린 답을 확신하는 모델은 행정 오류를 낳는다. ForecastBench에서 GPT-5.5·Claude Opus 4.8보다 높거나 동등 — 예측·예보 특화 파인튜닝의 기반 모델로 적합.
