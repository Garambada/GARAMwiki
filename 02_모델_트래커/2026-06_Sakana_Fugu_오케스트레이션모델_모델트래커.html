한 줄 요약 — Fugu가 무엇인가
🐡 핵심 정의
Sakana Fugu는 멀티에이전트 시스템처럼 행동하되 단일 모델처럼 보이는 오케스트레이션 AI다. 하나의 API 엔드포인트에 요청을 보내면 Fugu가 내부적으로 최적 모델 조합을 선택·위임·검증·종합한다. 사용자는 멀티에이전트의 복잡성 없이 프런티어 수준 성능을 얻는다. Fugu 자체도 언어 모델이며, 재귀적으로 자기 자신을 호출할 수 있다.
⚖️
두 가지 모델 — Fugu vs. Fugu Ultra
🐡 Fugu
STANDARD
고성능과 저지연(latency)의 균형. 일상 업무의 기본 선택. 코드 리뷰, 코딩 도구(Codex 연동), 챗봇·인터랙티브 서비스에 적합. 데이터·프라이버시·컴플라이언스 요건에 따라 특정 에이전트를 풀에서 제외하는 옵션 제공.
주요 사용 사례 코딩 · 코드 리뷰 · 챗봇 · 일상 업무 자동화
🐡⚡ Fugu Ultra
ULTRA · FRONTIER-MATCH
난이도 높은 다단계 문제에서 최고 품질의 답변을 위해 조정됨. 더 깊은 전문 에이전트 풀을 연동. 정확도와 깊이가 최우선인 작업에 최적화. Fable 5·Mythos Preview와 동급 성능이면서 수출통제 리스크 없음.
주요 사용 사례 AI 연구 · 논문 재현 · 사이버보안 분석 · 특허/문헌 조사 · 금융 시계열 예측

핵심 설계 원칙: 에이전트 풀은 완전히 교체 가능(swappable)하다. Fable 5·Mythos Preview처럼 수출통제로 접근이 차단된 모델은 풀에서 제외되고, Fugu가 자동으로 우회 라우팅한다. 새 프런티어 모델이 출시되면 풀에 추가된다 — 사용자는 아무 코드 변경 없이 최신 성능을 얻는다.

베타 사용자 500명 — 실사용 후기
"코드 리뷰에서 Fugu Ultra는 GPT-5.5보다 월등히 우수하다. 다른 도구들이 3건 정도 문제를 지적할 때 Fugu는 20건 이상을 찾아냈다. 이제 모든 리뷰를 Fugu로 돌린다."
— 소프트웨어 엔지니어 (코딩·코드리뷰)
"원시 출력 품질은 최상위 프런티어 모델과 동등하다. 더 놀라운 건 페르소나 안정성 — 장시간 세션에서 다른 모델들이 정체성을 잃어갈 때 Fugu는 흔들리지 않았다. 에이전트 프로덕트에서 이것은 벤치마크 점수보다 더 중요할 수 있다."
— 엔터프라이즈 플랫폼 기업 임원 (오케스트레이션 품질)
"단 하나의 지시로 Fugu가 정찰 → XSS/SQLi 검사 → 인증 리뷰 → 증거·재테스트 절차가 담긴 리포트까지, 보안 평가 전 과정을 자율 수행했다. 지정된 범위를 벗어나지 않았고, 파괴적 행동도 없었다."
— 사이버보안 엔지니어 (보안 평가 분석)

연구 기반 — ICLR 2026 발표 논문 2편
🔮
TRINITY
ICLR 2026 · arXiv 2512.04695
An Evolved LLM Coordinator. 진화 알고리즘으로 학습된 LLM 조율자. 여러 모델을 협조시켜 집합 지능을 이끌어내는 방법론의 기초 연구. Xu, Sun, Schwendeman, Nielsen, Cetin, Tang 공저.
🎼
Conductor
ICLR 2026 · arXiv 2512.04388
Learning to Orchestrate Agents in Natural Language. 자연어로 에이전트 오케스트레이션을 학습하는 방법론. "Conductor"가 에이전트들에게 자연어 지시를 내리고 결과를 조율. Nielsen, Cetin, Schwendeman, Sun, Xu, Tang 공저.

두 논문 모두 "학습된 오케스트레이션(learned orchestration)" 개념을 기반으로 한다 — 고정된 워크플로가 아니라 어떻게 위임하고 조율할지를 에이전트가 스스로 학습한다는 원리. 이것이 Fugu가 새 모델이 출시될 때마다 자동으로 성능이 향상되는 이유다.

🌏
지정학적 맥락 — AI 주권과 수출통제 대응

Sakana AI는 Fugu를 단순한 기술 제품이 아니라 지정학적 헷지(geopolitical hedge)로 명시적으로 포지셔닝한다. 2026년 6월 미국 정부의 수출통제로 Fable 5·Mythos Preview가 비미국인에게 차단된 사건을 직접 언급하며, 이를 출시 배경으로 제시한다.

핵심 주장: "AI 인프라, 금융, 행정을 단일 회사의 API에 의존하는 것은 물질적 취약성이다." 수출통제·외교정책·규제 변화로 하룻밤 새 접근이 차단될 수 있다. Fugu의 교체 가능한 에이전트 풀 구조는 이 리스크에 대한 구조적 답변이다.

🛡️ AI 주권 (AI Sovereignty) 설계 원칙
① 멀티벤더 풀: 특정 모델 접근 차단 시 자동 우회 라우팅
② 단일 API 추상화: 내부 교체가 사용자 코드에 영향 없음
③ 자체 모델 통합: 장기적으로 Sakana 자체 모델 비중 증가
④ 오픈 모델 지원: 오픈소스 모델도 풀에 통합 가능
→ 특정 국가·정부·기업의 정책 결정에 종속되지 않는 AI 인프라

01
AI 전략 패러다임 전환
스케일이 아닌 오케스트레이션 — 다음 프런티어
Fugu의 성공은 단일 거대 모델 스케일링 시대의 종언을 알린다. 어떤 모델을 쓰는가보다 어떻게 조율하는가가 성능을 결정하는 시대. Anthropic·OpenAI가 더 큰 모델을 만드는 동안, Sakana는 조율 방법을 학습시켰다. 어느 방향이 더 빠를지 모를 레이스가 시작됐다.
02
벤더 리스크 관리
AI 인프라 멀티벤더화 — 한국 기업 긴급 전략 과제
Fable 5·Mythos 수출통제 사건은 단일 AI 벤더 의존의 BCP(업무연속성) 리스크를 현실화했다. GARAMbada 사업 운영에서도 AI 인프라를 단일 API에 의존하는 것은 리스크다. 멀티벤더 API 추상화 레이어를 지금 설계해야 한다. Fugu 자체가 그 해법이 될 수도 있다.
03
에이전트 프로덕트 설계
페르소나 안정성 — 장시간 세션의 숨겨진 핵심 지표
베타 사용자가 언급한 "페르소나 안정성"은 새로운 중요 지표다. 벤치마크 점수가 같아도 장시간 에이전트 워크플로에서 정체성이 흔들리는 모델은 프로덕션에서 실패한다. 에이전트 제품 평가 시 장시간 세션 일관성을 반드시 테스트해야 한다.
04
비즈니스 모델 참고
단일 API 추상화 → SaaS 비즈니스 설계 원칙
Fugu의 사업 모델: 내부 복잡성(멀티에이전트)을 완전히 숨기고 단순한 단일 API를 판매. 사용자는 내부를 모른다 — 알 필요도 없다. AI 도구를 활용한 서비스 사업에도 동일 원칙 적용: 내부 오케스트레이션 복잡성을 고객에게 노출하지 말고 결과만 전달하라.
05
일본 AI 생태계
Sakana AI의 포지셔닝 — 집합지능 vs. 거대모델
Sakana AI는 David Chalmers·Jeff Clune 등 세계 최정상 AI 연구자들이 참여한 일본 기반 스타트업. 자연계 진화·집합 지능에서 영감을 얻은 고유 철학으로 차별화. 미국 빅테크의 스케일링과 다른 경로로 프런티어 성능에 도달하는 것을 증명한 사례. 한국 AI 스타트업에 참고가 되는 포지셔닝 전략.

즉시 활용 가능성
