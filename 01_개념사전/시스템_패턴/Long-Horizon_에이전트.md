# Long-Horizon 에이전트

## 한 줄 정의
> 수십 분~수 시간 동안 자율적으로 계획·행동·실패·수정을 반복하며 목표를 달성하는 AI 시스템.

## 왜 알아야 하나?
Sequoia Capital이 2026년을 "Long-Horizon Agent의 해"로 선언했다. 단순히 질문에 답하는 AI(Talker)와 달리, 실제로 **일을 완수하는 AI(Doer)**다. 이것이 가능해진 순간이 실용적 의미의 AGI 도달이며, 기업의 운영 방식과 소프트웨어 비즈니스 모델을 근본적으로 바꾼다.

## 핵심 메커니즘

```
[목표 입력]
    ↓
[계획 수립]
    ↓
[도구 사용 / 행동]  ←──────────────┐
    ↓                              │
[결과 평가]                        │ 실패 시
    ↓                              │ 방향 전환
[성공?] ──────── No ───────────────┘
    │
   Yes
    ↓
[목표 달성]
```

인간 전문가가 "머릿속에서 돌리는 루프"를 AI가 자동으로, 지치지 않고 실행한다.

## 기존 AI와의 결정적 차이

| 구분 | 기존 AI (Single-turn) | Long-Horizon Agent |
|------|----------------------|-------------------|
| 작동 방식 | 질문 → 답변 (1회) | 목표 → 자율 실행 (반복) |
| 시간 범위 | 수 초 | 수십 분 ~ 수 시간 |
| 실패 처리 | 멈춤 | 방향 전환 후 재시도 |
| 모호성 처리 | 어려움 | 가설 수립·검증으로 해결 |
| 사람 개입 | 매 단계 필요 | 목표 설정과 결과 검토만 |

## 3가지 필수 구성 요소

```
1. 기억(Memory)
   - 작업 중 맥락을 유지하고 이전 시도를 기억

2. 도구(Tools)
   - 웹 검색, 코드 실행, API 호출, 파일 읽기/쓰기 등

3. 하네스(Harness / Scaffolding)
   - 에이전트가 오래 작동할 수 있게 설계된 인프라
   - 메모리 핸드오프, 오류 복구, 자기 평가 루프 포함
```

## 비즈니스 임팩트

**가격 책정 패러다임 전환**:
- 기존: 시트 수 × 월정액 (도구 접근권)
- 신규: 완료한 작업 수 × 단가 (결과물 기반)

**조직 구조 변화**:
```
Before: IC → 직접 작업 수행
After:  IC → 에이전트에게 위임 → 결과 검토 및 방향 설정
```

**성능 성장 속도** (METR 측정):
- 약 7개월마다 처리 가능 작업 시간이 2배 증가
- 2028년: 하루치 작업, 2034년: 1년치 작업 처리 예상

## 주의할 함정

- **아직 실패한다**: 환각, 맥락 손실, 잘못된 방향으로 자신 있게 돌진하는 문제 존재. 고위험 업무에 Human-in-the-loop 필수
- **Human Attention이 새 병목**: 에이전트 여러 개를 동시에 돌릴수록 인간의 주의·감독이 새 제약이 됨 (Greg Brockman)
- **과도한 자동화 경계**: 모든 단계를 자동화하면 안 됨. "언제 인간이 개입해야 하나"를 명확히 설계해야 함
