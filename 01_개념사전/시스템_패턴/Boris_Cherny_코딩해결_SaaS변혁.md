# 코딩은 해결됐다 — Boris Cherny의 소프트웨어 변혁 프레임

## 한 줄 정의
> 에이전트 코딩이 성숙한 지금, 소프트웨어 개발은 '직접 코드를 쓰는 일'에서 '에이전트를 관리하는 일'로 전환됐다. 진짜 경쟁 우위는 모델이 아니라 **조직이 얼마나 빠르게 자신을 재편하느냐**에서 나온다.

## 왜 알아야 하나?
Boris Cherny는 Claude Code의 창시자이며, Anthropic에서 2026년 현재 가장 빠르게 10억 달러 연간 수익에 도달한 제품을 만든 사람이다. 그의 관찰은 단순한 예측이 아니라 **Anthropic 자신이 이미 살고 있는 현실**에서 나온다. C-레벨 의사결정자에게 이 내용은 "언젠가의 일"이 아닌 "지금 무엇을 바꿔야 하는가"의 문제다.

## 핵심 주장: 코딩은 해결됐다

| Boris의 현실 | 수치 |
|-------------|------|
| 2026년 직접 작성한 코드 | **0줄** |
| 하루 합병 PR 수 | 수십 개 (최대 150개/일) |
| 주요 작업 기기 | **스마트폰** |
| 운영 중인 동시 에이전트 | 수백 개 (야간 수천 개) |
| Claude Code 연간 수익 | **10억 달러+** (연환산) |

> "이건 내 개인적 버전의 '해결'이다. TypeScript + React 스택이 모델에게 가장 유리하다. 레거시 SAP나 C++ 임베디드 시스템은 아직 다르다. 하지만 **기다리면 된다. 답은 다음 모델이다.**" — Boris Cherny

## Claude Code 탄생 이야기

### Product Overhang(제품 잠복 기회) 전략
```
2024년 말 상태:
  IDE에서 Tab 키 한 번 → 코드 한 줄 생성 (Sonnet 3.5)
  → "이걸로 끝인가? 아니다. 모델은 이미 다음 단계가 가능하다"

Boris의 베팅:
  "6개월간 PMF 없을 것 알지만 우리는 다음 세대 모델을 위해 만든다"

변곡점:
  Opus 4 출시 (2025년 5월) → 지수 성장 시작
  이후 세대마다 곡선이 다시 꺾임 (4.5 → 4.6 → 4.7)
```

**핵심 교훈**: 지금 PMF가 없어도 **모델 능력 곡선의 다음 점에 먼저 서 있는 것**이 Product Overhang 전략이다.

## Loop — 미래의 기본 패턴

### 서브 에이전트보다 강력한 Loop
Boris의 핵심 작업 방식은 "여러 에이전트를 직접 지시"하는 게 아니라 **cron 기반 루프를 심어두고 스스로 돌아가게 하는 것**이다.

```
Loop 예시:
  → PR 모니터링 Loop: CI 실패 자동 수정, 자동 rebase
  → 품질 Loop: Flaky 테스트 자동 감지 및 수정
  → 인텔리전스 Loop: 30분마다 Twitter에서 Claude Code 피드백 수집 → 클러스터링 → 보고
  → 데이터 Loop: 데이터 변화 감지 → 자동 Slack 리포트
```

> **"Loop는 미래다."** — Boris Cherny

Anthropic Routines = Loop를 클라이언트에서 서버로 옮긴 것 (노트북 꺼도 지속 실행)

## 통才의 부상 — 조직 재편

### Claude Code 팀의 현실
팀 구성원 중 코드를 쓰는 사람:
- ✅ 엔지니어
- ✅ 엔지니어링 매니저
- ✅ **제품 매니저**
- ✅ **디자이너**
- ✅ **데이터 과학자**
- ✅ **재무 담당자**
- ✅ **사용자 리서처**

**모두가 코드를 쓴다.** 단, 각자 해당 분야의 전문가이기도 하다.

> 진정한 통才는 '프론트+백엔드+인프라 다 하는 엔지니어'가 아니라, **'도메인 전문가 + 코딩 능력을 갖춘 사람'**이다.

## SaaS 변혁 — 7 Powers 분석

Hamilton Helmer의 **'Seven Powers: 비즈니스 전략의 기반'** 프레임워크를 통한 AI 시대 해자 분석:

### AI가 무너뜨리는 해자 (위험)

| 해자 종류 | 기존 작동 원리 | AI의 무력화 방식 |
|----------|-------------|----------------|
| **전환 비용** | "이미 300개 워크플로가 Salesforce에 구성됨" | 모델이 마이그레이션 자동화 → 하룻밤에 이전 가능 |
| **프로세스 효력** | "우리 내부 프로세스는 따라올 수 없다" | Claude 4.7이 목표 설정 시 자동 hill-climb(최적화) |

### AI가 강화하는 해자 (안전)

| 해자 종류 | 이유 |
|----------|------|
| **네트워크 효과** | 사용자 증가로 강화되는 가치 — AI가 복제 불가 |
| **규모의 경제** | 규모에서 나오는 비용 우위 — 유지됨 |
| **독점 자원** | 특허, 면허, 독점 계약, 희귀 데이터 — 유지됨 |

> *"다음 10년, AI를 활용해 기존 시장을 뒤집는 스타트업 수는 과거 10년의 10배가 될 것이다. 대기업은 프로세스 전환과 내부 저항이 발목을 잡는다."* — Boris Cherny

## 인쇄술 유추 — 소프트웨어 민주화

| 시대 | 인쇄술 | 소프트웨어 |
|------|-------|---------|
| 도입 전 | 유럽 식자율 ~10-30% | 전문 개발자만 소프트웨어 제작 |
| 변혁 매개 | 구텐베르크 인쇄기 | AI 코딩 에이전트 |
| 도입 후 50년 | 유럽 출판물 수 1000년치 능가 | 소프트웨어 제작 비용 100분의 1 |
| 장기 결과 | 전 세계 식자율 90%+ | 소프트웨어 '발신' 능력 보편화 |
| 전문가 역할 | 작가 직업은 여전히 존재 | 소프트웨어 엔지니어 직업은 유지 |

**핵심 함의**: 최고의 회계 소프트웨어는 **엔지니어가 아닌 회계사**가 만들 것이다. 코딩은 쉬운 부분, 도메인 이해가 어려운 부분이기 때문이다.

## Anthropic 내부 — 조직 혁신이 진짜 해자

### 기술 격차 vs 조직 격차
| 항목 | 현실 |
|------|------|
| 모델 격차 | 내부 Mythos 사용, 하지만 미래에 동일 모델 외부 공개 |
| **조직 격차** | **이것이 진짜 차별화 포인트** |

### Anthropic 내부 현황 (2026)
- 모든 SQL: 모델이 생성
- 모든 제품 코드: 모델이 생성  
- 직원의 Claude가 Slack을 통해 다른 직원의 Claude와 소통
- **수기(手記) 코드 = 제로**

> *"모델은 누구나 쓸 수 있다. 하지만 조직 전체를 어떻게 재편하느냐가 경쟁 우위다."* — Boris Cherny

## C-레벨을 위한 전략 질문

1. **우리 조직에서 코딩 능력이 없어서 제품화 못 한 '도메인 인사이트'는 무엇인가?**
   → 이제 회계사, 영업 담당자, 운영 팀이 직접 소프트웨어를 만들 수 있다

2. **우리의 해자가 '전환 비용'과 '프로세스 효력'에 의존하고 있는가?**
   → 위험 신호. AI가 빠르게 무력화 중

3. **우리 조직에서 에이전트가 자율적으로 돌아가는 Loop는 몇 개나 있는가?**
   → 경쟁사 대비 Loop 수가 조직 AI화 지표

4. **Anthropic처럼 '모든 코드를 AI가 쓰는 조직'으로 전환하는 로드맵이 있는가?**
   → 기술보다 조직 전환이 더 오래 걸린다
