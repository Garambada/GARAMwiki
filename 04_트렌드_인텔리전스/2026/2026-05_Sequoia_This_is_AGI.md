## 3줄 요약

1. **AGI는 이미 도달했다** — Long-Horizon Agent가 "스스로 알아내는 능력"을 갖춘 순간, 실용적 의미의 AGI가 완성됐다.
2. **변곡점 3단계 완성** — 2022년 지식(ChatGPT) → 2024년 추론(o1) → 2026년 반복·실행(Long-Horizon Agent)으로 3단계가 채워졌다.
3. **AI는 "말하는 것"에서 "하는 것"으로** — 2026~2027년 AI 애플리케이션은 동료처럼 일을 실행하며, $10조 서비스 시장이 새 TAM이 된다.

## 핵심 프레임워크

### AGI의 기능적 정의

> *"AGI is the ability to figure things out. That's it."*
> — Pat Grady

Sequoia는 기술적 정의 대신 **비즈니스 관점의 기능적 정의**를 제시한다.

| 인간의 "알아내는 능력" | AI의 대응 요소 | 도달 시점 |
|---------------------|-------------|---------|
| 기초 지식 | 사전학습 (Pre-training) | 2022 (ChatGPT) |
| 지식 위의 추론 | 추론 시간 컴퓨트 (Inference-time compute) | 2024 (o1) |
| **반복하며 답을 찾는 능력** | **Long-Horizon Agent** | **2026 (Claude Code 등)** |

세 번째 재료가 채워지면서 AGI가 완성됐다는 주장.

---

### Long-Horizon Agent란 무엇인가?

기존 AI와의 결정적 차이:

| 기존 AI (Single forward pass) | Long-Horizon Agent |
|-----------------------------|--------------------|
| 질문에 답한다 | 일을 수행한다 |
| 한 번의 응답으로 끝 | 수십 분~수 시간 자율 작동 |
| 지시대로만 | 모호함 속에서 가설 수립·검증 |
| 실패 시 멈춤 | 실패 시 방향 전환 후 재시도 |

**실제 사례 (키노트 예시)**:
채용 에이전트에게 "DevRel 리드 찾아줘"라고 하면 → LinkedIn 검색 → YouTube 컨퍼런스 발표자 필터링 → Twitter 활동도 분석 → 최근 이직 신호 감지 → 맞춤 이메일 초안까지 **31분 만에 완료**.

## 비즈니스 임팩트

### 1. 새로운 TAM: $10조 서비스 시장

> 소프트웨어 산업이 한 번도 건드리지 못했던 서비스 수익(~$10 trillion)이 새 주소 가능 시장이 된다.

- 기존 소프트웨어는 "도구"를 팔았다 → 인간이 도구를 써서 일함
- Agent는 "일 자체"를 판다 → 결과물 기반 과금 가능

### 2. AI가 "고용 가능"해진 순간 = AGI

Sarah Guo의 기준: **"Can you hire it?"**

이미 고용 가능한 AI:

| 도메인 | 서비스 | 역할 |
|--------|--------|------|
| 의료 | OpenEvidence Deep Consult | 전문의 |
| 법률 | Harvey | 어소시에이트 |
| 보안 | XBOW | 펜테스터 |
| DevOps | Traversal | SRE |
| 세일즈 | Day AI | BDR/SE |
| 채용 | Juicebox | 리크루터 |

### 3. Talkers → Doers

```
2023~2024 AI: "Talkers"
→ 대화를 잘 하는 AI, 하루 몇 번 사용

2026~2027 AI: "Doers"
→ 동료처럼 일을 하는 AI, 하루 종일 여러 인스턴스 병렬 실행
→ IC로 일하다가 → 에이전트 팀을 관리하는 역할로 이동
```

### 4. The Diffusion Gap — 창업자·투자자 기회

```
[모델 진보 속도] ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                            ↑ 이 격차가 기회
[기업 도입 속도] ━━━━━━━
```

- 최신 모델 능력과 Fortune 500의 실제 도입 사이의 **격차(Diffusion Gap)**가 날마다 커진다
- 이 격차 = 애플리케이션 레이어 창업자의 기회
- 격차를 메우는 스타트업이 이긴다

## 창업자·투자자를 위한 MAD 프레임워크

Pat Grady의 조언:

| M | **M**oats from the customer back | 기술이 아닌 고객 워크플로우에서 해자 구축 |
|---|----------------------------------|---------------------------------------|
| A | Design for **A**ffordance | 에이전트의 새로운 능력을 활용하는 UI/UX 설계 |
| D | Exploit the **D**iffusion Gap | 대기업이 아직 못 도입한 영역 선점 |

## 우리에게 의미하는 것

**시간 지평**: 🔴 즉시 (지금 바로 전략 재검토 필요)

**핵심 질문**:

1. **우리 사업에서 "31분짜리 에이전트"가 대체할 수 있는 작업은 무엇인가?**
   → 반복적이고, 정보 수집 후 판단이 필요한 모든 작업

2. **우리가 파는 것이 "도구"인가, "일"인가?**
   → Agent 시대에는 결과물 기반 가격 책정(Outcome-based pricing) 전략 검토 필요

3. **Diffusion Gap에서 우리의 포지션은?**
   → 경쟁사 대비 AI 도입 속도가 앞서 있는가, 뒤처져 있는가

4. **에이전트를 "관리"하는 역량을 키우고 있는가?**
   → Greg Brockman: 에이전트가 자율 운영될 때 **인간의 주의(Human Attention)**가 새 병목
