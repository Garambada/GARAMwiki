01 — Architecture

4개의 기술 혁신 — 무엇이 새로운가

DeepSeek-V4는 단순 스케일업이 아니다. 1M 토큰 컨텍스트를 경제적으로 처리하기 위한 구조적 재설계.

V4 신규
CSA + HCA
Compressed Sparse Attention + Heavily Compressed Attention
하이브리드 어텐션 설계. CSA는 압축 비율 m으로 K/V를 압축한 뒤 top-k 엔트리에만 sparse attention 적용. HCA는 더 극단적인 압축으로 dense attention을 그룹 단위 처리. 중요한 토큰은 정밀하게, 나머지는 효율적으로.
KV Cache: V3.2 대비 90% 감소
FLOPs: V3.2 대비 27% 수준
V4 신규
mHC
Manifold-Constrained Hyper-Connections
레이어 간 잔차(residual) 연결을 리만 매니폴드 기하학적 제약 위에서 최적화. 깊은 네트워크에서 그래디언트 소실 억제, 학습 안정성 향상. 1.6T 규모에서 안정적 학습 가능하게 하는 핵심 혁신.
효과: 학습 안정성 향상, 수렴 속도 개선
V4 신규
Engram Memory
Engram Conditional Memory
생물학적 뇌의 기억 흔적(Engram) 개념 차용. 조건부 메모리 활성화로 장문 컨텍스트에서 관련 정보를 선택적으로 강화. 200K+ 토큰 범위에서 추론 품질 향상. 단순 어텐션만으로 잡기 어려운 장거리 의존성 포착.
효과: 장문 컨텍스트 추론 품질 향상
V3 계승·강화
MLA + MTP
Multi-Head Latent Attention + Multi-Token Prediction
MLA: 히든 상태를 소형 잠재 벡터로 압축 후 어텐션 계산 → KV Cache 추가 감소. MTP: 단일 forward pass에서 여러 토큰 동시 예측 → 추론 속도 향상. 256~384 전문가 중 6 routed + 1 shared expert 활성화.
MoE: 1.6T 파라미터, 토큰당 49B 활성
MoE 핵심 — "1.6T는 마케팅 숫자, 실제 계산은 49B"
총 파라미터: 1,600B  |  토큰당 활성: 49B  |  비율: 3%만 활성화
추론 비용은 49B 모델 수준 — 성능은 1.6T 규모
Roofline 분석: 실제 GPU 비용은 활성 파라미터 기준으로 계산. 49B BF16 ≈ 98GB → H100 2장으로 서빙 가능

05 — Assessment

강점 & 한계 — 솔직한 평가

✅ 강점
수학·정량 추론 최강 — Putnam 만점, HMMT 95%. 오픈소스 최초 프론티어 수준
코딩 성능 1위 — SWE-bench 80%+, Claude Pro·GPT-5.5 능가
1M 토큰 실용화 — CSA 덕분에 경제적 비용으로 장문 처리
원가 파괴 — 동급 성능 대비 50~80배 저렴한 API 단가
오픈 웨이트 (Apache 2.0) — 파인튜닝, 자체 호스팅, 상업적 사용 완전 자유
KV Cache 혁신 — 90% 감소로 자체 호스팅 GPU 비용도 대폭 절감
✗ 한계
과학 추론 2위 — GPQA Diamond에서 Gemini 3.1 Pro에 밀림 (85% vs 94.6%)
멀티모달 미지원 — 현재 텍스트 전용. 이미지·음성 처리 불가
풀 모델 자체 호스팅 난이도 — 1.6T 전체는 NVL72급 클러스터 필요
중국 기업 리스크 — 데이터 프라이버시, 지정학 리스크 존재
에이전트 생태계 미성숙 — Claude, GPT 대비 툴·플러그인 지원 부족
한국어 성능 불확실 — 영어·중국어 중심 학습, 한국어 품질 별도 검증 필요
06 — C-Level Strategy

지금 무엇을 결정해야 하는가

DeepSeek V4가 가져오는 전략적 선택지와 리스크.

즉시 검토
수학·코딩 워크로드 이전
현재 GPT/Claude로 처리 중인 수식 분석, 코드 생성, 정량 리포트 작업. DeepSeek V4-Pro 테스트 → 성능 동등 시 비용 50~80배 절감.
즉시 검토
대량 처리 파이프라인
분류, 추출, 요약 등 볼륨이 큰 비실시간 작업. V4-Flash ($0.07/M input) 사용 시 원가 구조 전면 재편 가능.
3개월 평가
오픈소스 파인튜닝 전략
Apache 2.0으로 도메인 특화 파인튜닝 후 자산화. 자체 LLM 모델을 경쟁 해자로 만드는 경로. CV < 0.4이면 자체 호스팅 검토.
3개월 평가
자체 호스팅 ROI 재계산
V4-Pro의 활성 파라미터 49B → H100 2장으로 서빙 가능. 기존 70B 자체 호스팅 비용 기준 대비 재산출 필요.
보류
과학 추론 중심 워크로드
GPQA 기준 Gemini 3.1 Pro가 10%p 우세. 의학·화학·물리 추론이 핵심인 작업은 전환 전 충분한 벤치마크 필요.
리스크 관리
민감 데이터 처리
중국 기업 데이터 정책 리스크. 개인정보·의료·금융 데이터는 자체 호스팅 전제로만 검토. API 버전은 민감 데이터 금지.
💡
경쟁사가 DeepSeek V4로 원가를 90% 낮춘다면?
LLM 비용이 더 이상 경쟁 우위가 아님. 원가 절감보다 컨텍스트 데이터·워크플로우 자동화 속도가 다음 해자. 지금 비용 감사해서 절감분을 에이전트 구축에 재투자.
⚖️
Claude/GPT vs DeepSeek — 언제 무엇을 쓰나?
수학·코딩·대용량 → DeepSeek V4. 과학 추론·멀티모달 → Gemini. 에이전트·툴 체인 → Claude/GPT. 단일 모델 전략은 2026년부터 비효율. 태스크 라우팅 레이어 구축 검토.
