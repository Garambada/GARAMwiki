# AI 도입 실패사례 모음

> 타산지석. 남의 실패에서 배우는 것이 가장 저렴한 학습.

## 실패 패턴 분류

### ❌ 패턴 1: 환각(Hallucination) 과신
**사례**: 법률 문서 자동 생성 → 존재하지 않는 판례 인용으로 소송 패배
**교훈**: LLM 출력은 반드시 사실 확인 단계 필요. 고위험 영역에서는 Human-in-the-loop 필수
**적용**: 우리 시스템에서 검증 단계 없이 자동 발송하는 프로세스가 있는지 점검

---

### ❌ 패턴 2: 비용 폭증
**사례**: 프롬프트 최적화 없이 GPT-4 도입 → 예상의 10배 API 비용 발생
**교훈**: 프로덕션 전 반드시 토큰 사용량 시뮬레이션. 캐싱·배치처리 설계 필수
**적용**: 도입 전 월 예상 비용 계산을 ROI 프레임워크로 검토

---

### ❌ 패턴 3: 사용자 저항
**사례**: 고객센터 직원 AI 도구 도입 → "내 일자리 뺏긴다" 거부감으로 활용률 10%
**교훈**: AI는 대체가 아닌 증강(Augmentation) 프레이밍. 직원 교육과 인센티브 설계 선행
**적용**: 도입 전 담당 팀과 충분한 소통과 Pilot 운영

---

### ❌ 패턴 4: 데이터 유출
**사례**: 직원이 내부 기밀 데이터를 ChatGPT에 입력 → 학습 데이터로 활용 우려
**교훈**: AI 사용 정책 및 가이드라인 먼저 수립. Enterprise 계약 또는 자체 호스팅 검토
**적용**: 우리 직원 AI 사용 정책이 있는가?

---

### ❌ 패턴 5: 과도한 자동화
**사례**: 고객 이메일 100% 자동 응답 → 복잡한 클레임을 엉뚱하게 처리 → 고객 이탈
**교훈**: 자동화 범위를 단계적으로 확장. 신뢰도 낮은 케이스는 인간에게 에스컬레이션
**적용**: 자동화 범위 설계 시 "언제 인간이 개입해야 하나" 명확히 정의
